Primeros legos de página web

Durante nuestra primera clase, revisamos los conceptos básicos de HTML (como el uso de ‘< >’ dentro del código) para, más tarde, aplicarlos al formar una página web.

Lo primero que hicimos fue abrir Visual Studio Code, donde, en un documento nuevo, escribimos Soy una hacker. Este documento lo guardamos como index.html para así crear un documento que cada vez que lo abramos, nos redirige a internet. En la página que se abre en internet, aparecerá lo que hayamos escrito dentro del documento de index.html; en este caso: soy una hacker.

Dentro de Visual Studio Code, siempre checar que has guardado el documento. Para asegurarnos de esto, picamos command S y automáticamente el punto blanco junto al nombre del documento (que quiere decir que no está guardado), cambia a ‘<>’ (equivalente ha guardado). Otra nota importante es que al abrir el documento en internet diga: file:// Esto indica que está guardado en nuestra computadora.

Para nuestras entregas, inicialmente, íbamos a usar GitHub ya que permite tener un dominio de página web. A pesar de que la Ibero tiene bloqueada esta página y fue imposible subir nuestro documento, hicimos una cuenta con los siguientes pasos:

Vinculamos un correo electrónico, una vez que teníamos la cuenta hicimos un ‘New Repository’, ‘Add Read Me’, ‘Create, ‘Add file’ o, en todo caso, ‘Create New File’.

En ese momento fue cuando hubo un error y no pudimos subir el documento. Así que entramos a HTML5 Up, un sitio que te permite usar plantillas de página web donde el HTML se puede modificar para ajustar dicha página a tus necesidades. Decidí escoger el template Phantom.

Este template lo abrimos en Visual Studio Code para modificarlo. El chiste era quitar todo lo que dijera Lorem Ipsum, cambiar títulos, subir imágenes y aprender a hacer links internos o externos.

Para links: a href=pg1.html es un link interno. En este caso, ‘pg1.html’ es el nombre de la página a la que se vincula. Esto lo hice con el primer artículo de la página. En ‘generics’, que muestra un template general para entradas, dupliqué el documento y cambié el nombre a ‘art1.html’. Después en index.html vinculé ‘art1.html’ (es un link interno) a la primera entrada; es decir, a esta que ahora leemos.

Para imágenes: Buscar una imagen, descargarla y guardarla dentro de la carpeta de ‘images’ con un nombre y el tipo de imagen (.png, .jpg). En index.html, o en cualquiera de las páginas, ubicar la imagen que quieres cambiar.

Ej: src=images/hilma.png. Aquí ‘hilma.png’ es la imagen que yo escogí e ‘images’ indica que está dentro de ese folder.

Seguimos modificando toda la página, esto incluye el homepage, el menú, los títulos y las entradas de cada generic page. Podíamos escoger el tamaño de letra, por ejemplo.

Para ver las modificaciones que hicimos en el código, es necesario abrir el archivo de index.html desde mi computadora y refrescar desde internet.
